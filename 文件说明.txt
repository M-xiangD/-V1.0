# 电池健康检测工具 - 文件说明

## 📁 项目文件说明

### 🔸 核心功能文件（必须保留）

| 文件 | 类型 | 用途 | 使用方法 |
|------|------|------|----------|
| `battery_health_checker.py` | Python 脚本 | **核心电池检测程序**，负责获取电池信息、计算健康度、生成报告 | 命令行运行：`python battery_health_checker.py` |
| `start_web_server.py` | Python 脚本 | **启动Web服务器**，显示电池健康报告，自动处理端口占用 | 命令行运行：`python start_web_server.py` |
| `run_web.pyw` | Python 脚本 | **双击运行版本**，无控制台窗口，自动处理端口占用 | 直接双击运行 |
| `run_web.bat` | 批处理文件 | **批处理启动版本**，带控制台窗口，显示运行过程 | 直接双击运行 |
| `index.html` | HTML 文件 | **Web界面**，显示电池健康数据和图表可视化 | 由服务器自动加载 |

### 🔸 辅助文件

| 文件 | 类型 | 用途 | 使用方法 |
|------|------|------|----------|
| `README.md` | 说明文档 | **使用说明**，包含功能介绍和使用方法 | 用文本编辑器打开查看 |
| `requirements.txt` | 依赖文件 | **记录Python依赖包**，用于安装必要的库 | 命令行运行：`pip install -r requirements.txt` |
| `build_exe.py` | Python 脚本 | **构建可执行文件**，将项目打包为单个EXE文件 | 命令行运行：`python build_exe.py` |

### 🔸 生成的文件（自动创建，可删除）

| 文件 | 类型 | 用途 | 特点 |
|------|------|------|------|
| `battery_health_report.json` | JSON 文件 | **电池健康报告**，包含详细的电池信息 | 每次运行都会更新 |
| `battery-report.html` | HTML 文件 | **Windows系统电池报告**，由 `powercfg` 命令生成 | 每次运行都会更新 |
| `BatteryHealthChecker.spec` | 配置文件 | **PyInstaller构建配置**，用于生成可执行文件 | 构建过程中自动生成 |

### 🔸 可执行文件（构建产物）

| 文件 | 类型 | 用途 | 使用方法 |
|------|------|------|----------|
| `BatteryHealthChecker.exe` | 可执行文件 | **独立运行版本**，无需安装Python和依赖 | 直接双击运行 |

## 📁 目录说明

| 目录 | 用途 | 说明 |
|------|------|------|
| `dist/` | 构建输出目录 | 存放生成的可执行文件和相关文件 |
| `build/` | 构建临时目录 | 存放构建过程中的临时文件，可删除 |
| `.venv/` | 虚拟环境目录 | 存放Python依赖包，可删除（需重新安装依赖） |
| `__pycache__/` | Python缓存目录 | 存放Python编译后的字节码，可删除 |

## 🚀 使用方法

### 🔸 方法1：使用可执行文件（推荐）
1. 打开 `dist/` 目录
2. 双击 `BatteryHealthChecker.exe` 运行
3. 无需安装任何依赖，直接使用

### 🔸 方法2：使用Python脚本
1. 安装依赖：`pip install -r requirements.txt`
2. 运行方式：
   - 双击 `run_web.pyw`（无控制台）
   - 双击 `run_web.bat`（带控制台）
   - 命令行：`python start_web_server.py`

### 🔸 方法3：仅生成电池报告
- 命令行：`python battery_health_checker.py`
- 报告将保存为 `battery_health_report.json`

## 🎯 功能特点

1. **全面的电池检测**：使用 `powercfg` 和 WMI 双重数据源
2. **详细的健康评估**：计算健康度、容量损失、使用建议
3. **直观的Web界面**：图表可视化，响应式设计
4. **智能端口管理**：自动检测和释放端口占用
5. **多种启动方式**：适应不同用户习惯
6. **独立可执行文件**：方便分享和使用

## 🔧 技术说明

- **开发语言**：Python 3.6+
- **支持系统**：Windows（使用了 `powercfg` 命令和 WMI）
- **核心依赖**：`wmi` 包（用于获取实时电池状态）
- **Web技术**：HTML5, CSS3, JavaScript, Chart.js
- **构建工具**：PyInstaller（用于生成可执行文件）

## 📝 注意事项

1. **首次运行**：可能需要几秒钟时间生成电池报告
2. **权限要求**：无需管理员权限，但需要能够运行 `powercfg` 命令
3. **网络连接**：无需网络连接，所有功能均在本地运行
4. **浏览器**：自动打开默认浏览器显示报告
5. **端口占用**：自动处理端口 8000 被占用的情况

## 🎉 快速开始

最简单的使用方法：
1. 打开 `dist/` 目录
2. 双击 `BatteryHealthChecker.exe`
3. 等待程序检测电池并打开浏览器
4. 在浏览器中查看详细的电池健康报告
